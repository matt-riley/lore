# Support

Need help? You're in the right place. 👋

## How to get help

**GitHub Issues** is the right place for:

- Bug reports — something is broken or behaving unexpectedly.
- Feature requests — something you'd like Lore to do.
- Questions about behaviour that isn't clearly documented.

Use the issue templates when filing bugs or feature requests — they help a lot.

---

## Before filing an issue

1. **Check the docs** — [README](README.md), [compatibility](docs/compatibility.md), [support matrix](docs/support-matrix.md).
2. **Run the test suite** — `npm test` catches harness regressions. `npm run validate-schema` catches config drift.
3. **Check existing issues** — your question or bug may already have an answer.

---

## What's in scope

| Type | Where |
|---|---|
| Bugs in supported surfaces | GitHub Issues |
| Feature requests | GitHub Issues (discuss first for larger changes) |
| Security vulnerabilities | See [SECURITY.md](SECURITY.md) — *not* public issues |
| Experimental surfaces | Best-effort — file an issue but expect slower triage |

---

## Response time

This is a small project. There are no SLAs. Issues on supported surfaces are prioritised; experimental surfaces are best-effort.

---

## Want to fix it yourself?

If you found a bug and want to contribute a fix, [CONTRIBUTING.md](CONTRIBUTING.md) has everything you need to get started. Bug fixes with a clear reproduction case are always welcome!
